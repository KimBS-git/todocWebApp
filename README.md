# 토닥 (Todoc)

반려동물을 위한 동물병원 예약 모바일 앱인 토닥의 웹 애플리케이션 버전입니다.

---

## 배포

[todoc_web](https://todocwebapp.vercel.app/)

---

## 1. 프로젝트 구조

todoc_web_app/
├── api/
│   └── config.js          # Vercel Serverless: KAKAO_APP_KEY JSON 응답

│   ├── index.css          # 홈 전용
│   ├── search.css         # 병원검색 전용
│   ├── book.css           # 예약내역 전용
│   └── mypage.css         # 마이페이지 전용
├── html/
│   ├── index.html         # 홈(로그인/회원가입 + 대시보드)
│   ├── search.html        # 병원검색(카카오맵)
│   ├── book.html          # 예약내역
│   └── mypage.html        # 마이페이지
├── js/
│   ├── common.js          # 카카오 설정 로드, 스토리지, 로그인/회원가입, 기본 admin 시드
│   ├── index.js           # 홈 렌더링(예약·반려동물 카드), 로그인 부트스트랩
│   ├── search.js          # 병원검색·지도·예약 모달
│   ├── book.js            # 예약내역 탭·카드·후기
│   ├── mypage.js          # 프로필·반려동물·로그아웃
│   └── app.js             # (참고) SPA 안내용, HTML에서 미사용

├── vercel.json            # rewrites/redirects, search.js 캐시 헤더

## 2. 페이지별 주요 기능

### 홈 (`html/index.html` + `js/index.js`)

**역할:** 비로그인 시 로그인/회원가입, 로그인 후 홈 대시보드.

- **가상 지도 영역:** 실제 카카오 지도 API 없이 CSS로 맵 느낌의 장식 블록만 표시합니다.
- **「지도에서 병원 찾기」:** 병원 검색 페이지(`search.html`)로 이동하는 버튼입니다.
- **빠른 서비스:** 미용·호텔·동물카페·용품샵 네 가지를 가로 한 줄·정사각형 타일로 보여 줍니다.
- **다가오는 예약 / 이전 예약** 요약 카드로 예약 현황을 보여 줍니다.
- **나의 반려동물:** 마이페이지와 같은 사용자 반려동물 데이터를 카드로 가로 스크롤하며, **전체보기**는 마이페이지로 연결됩니다.

세션이 있으면 로그인/회원가입 화면을 숨기고 홈 대시보드만 그립니다. 

---

### 병원검색 (`html/search.html` + `js/search.js`)

**역할:** 카카오맵 SDK를 동적으로 불러와 지도를 띄우고, 장소(Places) 키워드 검색으로 병원 목록·마커·예약 모달을 처리합니다.

- 현재 위치, 지역 검색을 통해 주변 동물병원을 조회 가능합니다.
- 조회된 동물병원의 정보를 볼 수 있고 예약이 가능합니다.
- 검색창·칩 필터·현위치 버튼 등과 연동되어 목록과 마커를 갱신합니다.
- 예약날짜는 최솟값을 오늘로 설정되었고 연락처는 유효성 검사를 통해 형식에 맞춰 입력해야 합니다.

---

### 예약내역 (`html/book.html` + `js/book.js`)

**역할:** **예정된 예약**과 **지난 예약** 탭, 카드 목록, 상세·취소·후기·재예약 등 예약 관련 화면을 담당합니다.

- 예약을 현재 시각 기준으로 나누고, 예정 예약은 가까운 날짜순, 지난 예약은 최근순으로 정렬합니다.

---

### 마이페이지 (`html/mypage.html` + `js/mypage.js`)

**역할:** 프로필 표시, 반려동물 목록(추가·수정·삭제), 프로필 수정 모달, 로그아웃입니다.

- 반려동물이 없으면 안내 문구와 **추가** 버튼만, 있으면 목록 행과 **추가** 버튼을 보여 줍니다.
- **내 정보 관리** 영역에서 프로필 수정과 로그아웃으로 이어집니다.

---

### 공통 (`js/common.js`)

**역할:** 서버에서 카카오 앱 키를 받아 쓰기 위한 설정 로드, `localStorage` 기반 사용자·세션, 로그인/회원가입, 그리고 **기본 관리자 계정**이 없을 때만 생성하는 시드(데모용 지난 예약 한 건 포함)입니다.

- 사용자 저장소가 비어 있으면 빈 객체로 초기화한 뒤, 관리자 계정 보정 로직을 실행합니다.
- 현재 로그인한 사용자는 세션과 사용자 테이블을 합쳐 조회하고, 변경 사항은 같은 사용자 키로 다시 저장합니다.

---

### 배포 API (`api/config.js`)

**역할:** Vercel 환경 변수에 넣은 카카오 앱 키를 JSON으로 내려주어, 브라우저에서 카카오맵 초기화에 사용합니다.

- 응답은 JSON이며, 캐시되지 않도록 헤더를 둡니다.

## 3. 기술 스택

| 구분 | 사용 기술 |
|------|-----------|
| 마크업·스타일 | HTML5, CSS3 (페이지별 스타일시트 분리) |
| 스크립트 | 바닐라 JavaScript (ES6+), 모듈 번들러 없음 |
| 라우팅 | MPA — 페이지마다 별도 HTML, 링크로 이동 |
| 데이터 저장 | `localStorage`(사용자·세션·예약·반려동물), `sessionStorage`(검색 페이지 위치 캐시 등) |
| 지도·장소 | 카카오맵 JavaScript API, Places, `navigator.geolocation` |
| 폰트 | Pretendard (jsDelivr CDN) |
| 배포·백엔드 | Vercel Serverless Functions (`/api/config`), `vercel.json`(rewrites·redirects·캐시 헤더) |
